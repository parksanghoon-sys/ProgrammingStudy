## Why Clean Code ?

SW는 한번 작성되면 최소 10번 이상 읽히게 된다. 그래서 읽기 편하게 작성해야 한다.

우리의 주 업무는 남이 작성된 코드를 읽고, 수정하고, 기능을 추가하는 일을 하게 된다.

* 절차지향
  * 데이터가 있고 각 데이터를 공유해서 함수를 사용한다.
  * 데이터에 변경이 있으면 각 함수들에 영향이 가게된다.
* 객체지향
  * 데이터를 사용하는 기능은 맵핑이되어 객체에 변경이 잇어도 구체적인 모습을 다른 객체가 모두 알면 안된다.
  * 외부에 노출된 인터페이스가 변경되지 않는다면 프로시져를 실행하느데 필요한 만큼의 데이터만 가짐

## 객체

객체를 볼때 기능으로 봐야한다.

클래스 이름은 무엇으로 정의해야한다, 어떻게 정의하지 말아야한다.

객체는 역할을 가지며, 역할은 관련된 책임의 집합이다.

## Interface

상속은 재사용을 위함이 아니다 오히려 긴밀한 연결을 나타낸것이다. interface 기반으로 구현시에 이 인터페이스를 기반으로 사용중인 서비스에 변경 즉 수정이 없어 행동을 이어서 사용할 수있겟되는것이다, 이것이 재사용이다. 상속의 super 재사용은 좋은 방식이 아니다. 복잡한 결합도를 가진다.
