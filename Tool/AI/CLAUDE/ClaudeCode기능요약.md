## 개요

작성자는 Claude Code를 일상적으로 활용하면서 얻은 경험과 모범사례를 정리했어요. 개인 프로젝트는 물론 조직 단위 레포지토리에서도 쓰면서 유용했던 기능, 워크플로우, 제약 등을 정리한 레퍼런스 성격 글이고, 다양한 고급 기능을 설명합니다. ([blog.sshh.io](https://blog.sshh.io/p/how-i-use-every-claude-code-feature "How I Use Every Claude Code Feature - by Shrivu Shankar"))

## 주요 권고사항 & 통찰

* 너무 많은 커맨드/슬래시 명령을 만드는 것보다는 간결하고 직관적인 인터페이스를 유지하자는 관점.
* 문맥(token) 사용량을 잘 관리하되, 자동 압축(compact) 기능만 의존하지 말고 필요하면 명시적 초기화(`/clear`) + 재구성 방식 병행
* 에이전트가 자동 생성한 PR 등 산출물에 대해서도 사람이 검토하는 절차가 여전히 중요
* 스크립팅 가능한 구조(SKILLS / SDK / Hooks / GHA 등)를 적극 활용하면 단순히 “에이전트에게 물어보기” 수준을 넘어서 “CI/CD 파이프라인 일부로 Claude를 포함”할 수 있다
* 문서를 너무 장황하게 만드는 것보다는 중요한 규칙이나 경계(guardrails)를 간결하게 남기는 것이 더 효과적
